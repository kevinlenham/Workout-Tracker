# AGENTS.md

## Agent skills

### Issue tracker

Issues are tracked as GitHub Issues in this repo (via the `gh` CLI). External PRs are not treated as a triage surface. See `docs/agents/issue-tracker.md`.

### Triage labels

Uses the canonical label vocabulary as-is (needs-triage, needs-info, ready-for-agent, ready-for-human, wontfix). See `docs/agents/triage-labels.md`.

### Domain docs

Single-context layout — `CONTEXT.md` and `docs/adr/` at the repo root. See `docs/agents/domain.md`.

### Design context

Strategic and visual design decisions live in `PRODUCT.md` (register: product; personality:
calm & focused; anti-references: SaaS dashboards, gamified fitness apps, bare spreadsheets) and
`DESIGN.md` (the "Quiet Logbook" token system: flat tonal-layered surfaces, a single rationed
Quiet Teal accent, no shadows anywhere). Read both before any UI work via the `impeccable` skill.
